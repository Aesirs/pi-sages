# Sages

Web research providers for pi. This pi package registers tools that let the model search, crawl, and extract web content through a deterministic priority-based provider selection system. No more provider guessing — the highest-priority available sage is selected automatically.

## Architecture

### Provider Selection Flow

1. **SageSelector** holds a global priority list (default: `tavily > brave > firecrawl > exa > duckduckgo`)
2. When a tool executes, `selector.select(capability)` walks the list and picks the **first sage that:**
   - Has the requested `SageCapability` (`search`, `crawl`, or `extract`) in its profile
   - Is **configured** (has an API key or needs no key)
3. If none are available, it throws immediately with an availability breakdown — no failed API calls
4. The model never sees a `provider` parameter; it can't choose wrong

### Key Concepts

| Term | Meaning |
|---|---|
| **Sage** | A web provider (Firecrawl, Tavily, etc.) — has a profile, capabilities, needs a key or not |
| **SageCapability** | What a sage can do: `search`, `crawl`, `extract` |
| **SageProfile** | Static metadata: id, label, capabilities, needsKey |
| **Config** | API key configuration — stored in sages.json via CredentialStore or via env var |
| **SageSelector** | Priority list determining which sage is called first per capability |
| **CredentialStore** | Auth manager — bridges SageConfigStore (sages.json) and env vars |
| **SageResolver** | Factory that instantiates a sage if a valid config exists |

### Tools (what the model calls)

| Tool | Capability | Description |
|---|---|---|
| `search` | search | Search the web for current info, docs, facts |
| `crawl` | crawl | Crawl a URL to extract full page markdown |
| `extract` | extract | Extract structured content from specific URLs |
| `multiSearch` | search | Multi-provider search across all configured sages, deduplicated |

### Command

| Command | Purpose |
|---|---|
| `/sages` | Interactive menu: view status, manage API keys, set priority |
| `/sages set-key <sage> <key>` | Non-interactive key binding |
| `/sages clear-key <sage>` | Non-interactive key removal |
| `/sages priority <sage1> <sage2> ...` | Non-interactive priority override |

### Environment Variables

| Variable | Purpose |
|---|---|
| `SAGE_PRIORITY` | Override default priority. Comma-separated list or JSON array |
| `<SAGE>_API_KEY` | Per-sage API key (e.g. `TAVILY_API_KEY`, `FIRECRAWL_API_KEY`) |

## Adding a New Sage

1. Create `src/providers/<name>.ts` extending the `Sage` abstract class
2. Define a static `metadata: SageProfile` with id, label, capabilities, needsKey
3. Implement `search()`, and optionally `crawl()` / `extract()`
4. Export from `src/providers/index.ts`
5. Register the sage class in `src/registry.ts` `SAGE_CLASSES` array
6. Optionally add the new id to the default `DEFAULT_PRIORITY` in `src/selector.ts`
