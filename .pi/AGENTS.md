# Sages

A guild of sages for web research. This pi package registers tools that let the model search, crawl, and extract web content through a deterministic priority-based provider selection system. No more provider guessing — the highest-priority available sage is summoned automatically.

## Architecture

### Provider Selection Flow

1. **SummonOrder** holds a global priority list (default: `tavily > brave > firecrawl > exa > duckduckgo`)
2. When a tool executes, `order.select(art)` walks the list and picks the **first sage that:**
   - Has the requested `SageArt` (`search`, `crawl`, or `extract`) in its profile
   - Is **bound** (configured with an API key or needs no key)
3. If none are available, it throws immediately with an availability breakdown — no failed API calls
4. The model never sees a `provider` parameter; it can't choose wrong

### Key Concepts

| Term | Meaning |
|---|---|
| **Sage** | A web provider (Firecrawl, Tavily, etc.) — has a profile, practices arts, needs a contract or not |
| **SageArt** | What a sage can do: `search`, `crawl`, `extract` |
| **SageProfile** | Static metadata: id, label, capabilities, needsKey |
| **Contract** | API key binding — stored in auth.json via VaultKeeper or via env var |
| **SummonOrder** | Priority list determining which sage is called first per art |
| **VaultKeeper** | Auth manager — bridges AuthStorage (auth.json) and env vars |
| **Summoner** | Factory that instantiates a sage if a valid contract exists |

### Tools (what the model calls)

| Tool | Art | Description |
|---|---|---|
| `scry` | search | Search the web for current info, docs, facts |
| `delve` | crawl | Crawl a URL to extract full page markdown |
| `decipher` | extract | Extract structured content from specific URLs |
| `commune` | search | Multi-provider search across all bound sages, deduplicated |

### Command

| Command | Purpose |
|---|---|
| `/guild` | Interactive guild hall: inspect council, sign/break contracts, set summon order |
| `/guild set-key <sage> <key>` | Non-interactive key binding |
| `/guild clear-key <sage>` | Non-interactive key removal |
| `/guild priority <sage1> <sage2> ...` | Non-interactive priority override |

### Environment Variables

| Variable | Purpose |
|---|---|
| `SAGE_SUMMON_ORDER` | Override default priority. Comma-separated list or JSON array |
| `<SAGE>_API_KEY` | Per-sage API key (e.g. `TAVILY_API_KEY`, `FIRECRAWL_API_KEY`) |

## Adding a New Sage

1. Create `src/providers/<name>.ts` extending the `Sage` abstract class
2. Define a static `metadata: SageProfile` with id, label, capabilities, needsKey
3. Implement `search()`, and optionally `crawl()` / `extract()`
4. Export from `src/providers/index.ts`
5. Register the sage class in `src/registry.ts` `SAGE_CLASSES` array
6. Optionally add the new id to the default `DEFAULT_SUMMON_ORDER` in `src/selector.ts`
